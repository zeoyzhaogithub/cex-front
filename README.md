### 订单薄：

#### 基础概念

订单簿（Order Book）
bids：买盘（绿色部分）
asks：卖盘（红色部分）

反映了当前交易者的意愿，但还未实际成交

盘口（plate）通常由以下几部分组成

1. 价格（报价）
2. 数量：当前价格的代币交易数量
3. 合计数量：交易数量到当前价格的合计
4. 最近成交价格（相较上次涨或跌）

#### 逻辑实现

以买盘（bids，绿色表示）为例：

1. 价格：从上到下，由高到低排序
2. 交易(BTC)数量
3. 交易合计的计算

 数据结构：
{
    asks: [ { price:xx, amount:xx }, ... ],
    bids: [ { price:xx, amount:xx }, ... ],
}

交易的合规计算
#### 逻辑实现

以买盘（绿色表示）为例：

3. 交易合计的计算，计算当前价格往上的所有交易数量的总和举例：
[22 + 1096 = 1118], [1118 + 22 = 1140]

深色部分表示，当前交易数量占总数量的百分比
item.totalAmount / plate.bidTotal * 100%

# 指标

用于辅助投资者分析资产的价格走势，指标的计算来自于各种数学公式，可以开发自己的交易指标。

- 移动平均线（MA）
- 判断整体价格走势
- 布林通道（Bollinger Bands）
- 常用于判断压力线、支撑线，辅助判断交易的进出场时机
